1. Project Goal

We are preparing a new version of the TEJESTA Shopify store before making changes to the live site.

The main goal is to organize the catalog so the store becomes clearer, easier to maintain, and closer to a premium eyewear experience inspired by Matsuda:

  • One product = one eyewear model/style.
  • Color and lens variations are managed as variants inside the product.
  • The variant option is called Colorway.
  • Collection pages stop showing many duplicated products by color.
  • Each product page concentrates images, description, variants, SEO, and commercial data in one place.

5. How Colorway Images Will Work

When old separate products are merged into a single new product, we need to avoid mixing images from different colorways in the same gallery.

The defined solution is that each Colorway variant will have its own image gallery through a custom Shopify field.

Instead of creating rigid fields such as Front, Side, and Diagonal, we will use a flexible gallery field per variant. This means each colorway can have three images when that is enough, but it can also have more images if a product needs them.

Behavior

  • Each product will continue using Colorway variants.
  • Each variant will have its own list of images.
  • The theme will show the image gallery that belongs to the selected colorway.
  • The team will be able to add more photos from the backend when needed.

Recommended Image Order

To keep consistency, we recommend this order:

  1. Front
  2. Side
  3. Diagonal
  4. Detail or extra images, when needed

Benefits

  • Each colorway controls its own images.
  • The gallery does not depend on filenames or ALT text.
  • The system is cleaner and more scalable long term.
  • Products are not limited to only three photos.
  • It supports a clearer and more premium visual experience.

Consideration

This solution requires theme customization so the theme can read the images assigned to each variant, but it is the cleanest option for the new catalog structure.

6. Visual Point To Review: Selector With Many Variants

On the product page, the colorway selector works well when there are only a few variants. For example, products with four colorways look clean and easy to use.

But in models with many variants, such as Araki, the thumbnails do not fit well in the available space and the selector looks too compressed.

We should review this with TEJESTA before closing the product page design.

Options we could consider:

  • Keep small thumbnails but add horizontal scroll.
  • Show fewer visible thumbnails and open a full selector on click.
  • Use a multi-row layout.
  • Switch to a list/dropdown selector for models with many variants.
  • Use larger thumbnails on desktop and a more compact solution on mobile.

The important point is deciding which experience TEJESTA prefers when a model has many colorways.

7. Pending Point: Sold-Out Variants

We need to decide what happens when a variant/colorway is sold out.

Options:

  • Hide it from the website while it is out of stock.
  • Keep it visible as a thumbnail, but with a slight fadeout or clear sold-out state.

Keeping it visible helps show the full colorway range and can generate interest, but it can also frustrate users if they try to select something they cannot buy.

Hiding it simplifies the shopping experience, but it can make a model look like it has fewer options than it actually does.

Decision needed from TEJESTA: hide sold-out colorways, or show them with a clear visual state.

8. Pending Point: Product Title On PDP

Right now, the product page shows the product/model name.

We could make the title also show the selected colorway when a variant is selected.

Example:

  • Current: Araki
  • Possible: Araki - Piano
  • Possible: Araki - Havana

This can help the user understand which colorway they are viewing, especially if the gallery and images change when a variant is selected.

Decision needed from TEJESTA: keep a cleaner title with only the model, or show model + colorway on the PDP.

9. Editorial Blocks On PDP

On references like Matsuda, after the product gallery there are editorial hero blocks with images, text, and links.

These blocks should not depend on the variant/colorway. The cleanest approach is to treat them as product, family, or collection content, not as variant-specific content.

Recommendation:

  • Keep one common PDP base for eyewear.
  • Add reusable editorial blocks below the gallery.
  • Manage those blocks with metafields or metaobjects, so they can be shared across products in the same family.
  • Avoid creating a different template for every single model, except for special cases.

When separate templates could make sense:

  • Sunglasses vs Optical, if the structure changes.
  • Accessories or apparel.
  • Special collaborations.
  • Landing-style products with very specific storytelling.

For normal eyewear models, it is better to have one flexible main template and control editorial content from Shopify.

10. Pending Point: Collection Filters

We still need to define the filtering system.

It is not enough to decide what filters the shopper will see. We also need to decide what internal information will feed those filters:

  • Tags.
  • Product type.
  • Metafields.
  • Collections.
  • Specific internal product fields.

Possible filters to review:

  • Sunglasses / Optical.
  • Shape.
  • Frame material.
  • Frame color.
  • Lens color.
  • Fit.
  • Availability.
  • Price.
  • Collection.

We also need to decide how filters will be displayed visually: sidebar filters, top filters, buttons, dropdowns, or a combination depending on desktop/mobile.

This remains open before closing the collection design.

Important Note

The development site is a working environment. It is useful for reviewing and approving the new structure, but it should not be considered the final production version yet.

Product image metafields and final product collections should be completed inside the real/live Shopify store after the new theme has been imported there, even if that theme remains unpublished during setup. This avoids duplicating manual work in development and ensures the metafield images and collections are connected to the final imported products and variants.
